## 一、本次修复的Bug列表

### 1. B30管理 - 添加成绩选歌跳转问题
**问题描述**：选完歌曲后未能返回添加成绩页面，而是跳转到了容错计算页面

**根本原因**：`songs.vue` 使用 `onMounted` 获取页面参数，在 App 端时机不对

**解决方案**：改用 UniApp 的 `onLoad` 生命周期获取 `options`

```typescript
// 错误写法 ❌
onMounted(() => {
  const pages = getCurrentPages()
  const currentPage = pages[pages.length - 1]
  const options = currentPage.options as any
  if (options && options.from) {
    fromPage.value = options.from
  }
})

// 正确写法 ✅
import { onLoad } from '@dcloudio/uni-app'

onLoad((options: any) => {
  if (options && options.from) {
    fromPage.value = options.from
    console.log('从页面进入:', fromPage.value)
  }
  loadSongs()
})
```

### 5. H5端容错计算页面跳转问题
**问题描述**：选择歌曲后错误跳转回歌曲列表，而不是返回容错计算页面

**根本原因**：storage 操作是同步的，在数据写入完成前就执行了页面跳转

**解决方案**：将 storage 操作改为异步方式（使用 Promise），确保数据写入完成后再执行页面跳转

```typescript
// 错误写法 ❌ - 同步写入未完成就跳转
uni.setStorage({ key: 'recent_song', data: songInfo })
uni.navigateBack()

// 正确写法 ✅ - 异步等待写入完成
await new Promise((resolve, reject) => {
  uni.setStorage({
    key: 'recent_song',
    data: songInfo,
    success: resolve,
    fail: reject
  })
})
uni.navigateBack()
```

### 6. 页面生命周期钩子问题
**问题描述**：`Cannot read property 'name' of undefined` 错误

**根本原因**：在 `onShow` 生命周期中访问数据，但数据尚未加载完成

**解决方案**：使用 UniApp 组合式 API 的 `onPageShow` 替代选项式 API 的 `onShow`

```typescript
// 错误写法 ❌ - onShow 可能过早执行
import { onShow } from '@dcloudio/uni-app'
onShow(() => {
  const song = uni.getStorageSync('recent_song') // 可能为空
})

// 正确写法 ✅ - 使用 onPageShow 或添加数据检查
import { onPageShow } from '@dcloudio/uni-app'
onPageShow(() => {
  const song = uni.getStorageSync('recent_song')
  if (song && song.name) {
    // 安全使用 song
  }
})
```

## 二、IIFE 打包错误解决方案

### 错误信息
```
Invalid value "iife" for option "output.format" - UMD and IIFE output formats are not supported for code-splitting builds.
```

### 问题分析
UniApp Android 打包时使用 IIFE 格式，但代码中使用了动态 `import()` 导致代码分割，两者冲突。

### 最终解决方案：使用条件编译

#### 1. `cloudbase.ts` - 云开发代码条件编译
```typescript
// #ifdef H5
import cloudbase from "@cloudbase/js-sdk";
// #endif

// #ifdef H5
export const auth = cloudbase.auth({ persistence: "local" });
// #endif

// #ifdef H5
export async function initCloudBase() { ... }
// #endif

// #ifndef H5
export async function initCloudBase() { return false; }
export function checkEnvironment() { return false; }
// #endif
```

#### 2. `show-captcha.vue` - 验证码组件
```typescript
// #ifdef H5
import { auth } from '../utils/cloudbase'
// #endif

// #ifdef H5
const handleSubmit = async () => { ... }
// #endif

// #ifdef H5
uni.$on('CAPTCHA_DATA_CHANGE', captchaDataHandler)
// #endif
```

#### 3. `App.vue` - 主入口
```typescript
// #ifdef H5
import { initCloudBase, checkEnvironment } from "./utils/cloudbase";
// #endif

onLaunch(async () => {
  // #ifdef H5
  if (checkEnvironment()) {
    await initCloudBase();
  }
  // #endif
});
```

#### 4. `vite.config.ts` - 构建配置优化
```typescript
export default defineConfig({
  plugins: [uni()],
  build: {
    target: 'es2015',
    cssCodeSplit: false,
    rollupOptions: {
      output: {
        inlineDynamicImports: true
      }
    }
  }
});
```

### 核心思路
- **H5 端**：正常使用云开发功能
- **App 端**：通过 `#ifdef H5` 排除云开发代码，避免动态 `import()` 导致的代码分割
- **兜底函数**：App 端提供空的 `checkEnvironment()` 等函数，避免编译报错

---

## 三、UniApp 条件编译语法

| 语法 | 说明 |
|------|------|
| `// #ifdef H5` | 只有 H5 端编译 |
| `// #ifndef H5` | 只有非 H5 端编译 |
| `// #endif` | 结束条件编译块 |

适用场景：
- 云开发功能（H5专属）
- 平台特有API（如文件系统的 FileReader vs uni.downloadFile）
- 需要避免代码分割的动态导入

---

## 四、经验总结

1. **页面参数获取**：UniApp 中获取页面参数应使用 `onLoad(options)` 而非 `onMounted`

2. **云开发兼容性**：云开发 SDK 只能用于 H5 端，App 端必须使用条件编译排除

3. **IIFE 错误根源**：动态 `import()` 会产生代码分割，与 IIFE 格式冲突

4. **条件编译优势**：
   - 保持代码统一管理
   - 按平台选择性编译
   - 不影响 H5 端功能

5. **App 端导出回退**：App 端文件系统 API 限制多，失败时可回退到剪贴板
